# 4P-Score — オリジナル・エントリーシグナルシステム

## 概要

「4つの鉄板エントリーパターン解説マニュアル」の手法を**ダウ理論ベースで忠実に実装した**
オリジナルのシグナル検出システムです。

従来EA（`FX4Pattern_EA.mq5`）はMAの傾きだけでトレンドを近似していましたが、
本システムはマニュアルの核心である**「高値切り下げ＋安値更新」（ダウ理論の転換）を
実際のスイング高値・安値から検出**し、さらにエントリー前に必ず確認すべき
**要素①（日足合致）・要素②（反対勢力）を点数化**してシグナルを厳選します。

```
mt5/
├── FX4Pattern_EA.mq5       # 従来版EA（MAスロープ・パターン①のみ）
├── FX4Pattern_Signal.mq5   # ★本システム（ダウ理論・全パターン・採点制）
└── README_SIGNAL.md        # このファイル
```

## 従来版との違い

| 項目 | FX4Pattern_EA（従来版） | 4P-Score（本システム） |
|------|------------------------|----------------------|
| トレンド判定 | MA20/80/480の傾き | ダウ理論（スイング高値・安値の切り上げ/切り下げ） |
| 転換検出 | MAスロープの反転 | 高値切り下げ＋直近安値の実ブレイク（マニュアル準拠） |
| 監視パターン | ①のみ | ①②③＋④-B情報通知 |
| 要素①日足合致 | ON/OFFフィルター | 採点（+20点）・逆行は却下 |
| 要素②反対勢力 | なし | 上位足スイングまでの距離をATR比で採点（近すぎは却下） |
| シグナル選別 | なし（条件成立で即） | 0〜100点採点 → S/A/Bグレード |
| SL位置 | ATR×2固定 | 直近押し安値・戻り高値の向こう側（マニュアル準拠） |
| スマホ通知 | なし | プッシュ通知対応（MT5モバイルアプリ） |
| 自動売買 | 常時 | デフォルトOFF（シグナル専用機）・Sグレードのみ自動も可 |

## スコアリングの仕組み

| 項目 | 配点 | 内容 |
|------|------|------|
| 基本トリガー | 40点 | 執行足でダウ理論の転換が確定（高値切り下げ＋安値更新 or その逆） |
| 要素① 日足合致 | +20点 | 日足が同方向。**逆行（戦争状態）は問答無用で却下**。日足レンジは+5点 |
| 要素② 反対勢力 | +20点 | 直下のサポート/直上のレジスタンス（上位足スイング・日足MA）までの距離。<br>ATR×3以上=+20 / ATR×2以上=+10 / ATR×1未満=**却下** |
| MA整列 | +10点 | レート・執行足MA・上位足MAがシグナル方向に整列＋上位足MAが同方向に傾斜 |
| 押し・戻りの深さ | +10点 | 直前波の30〜70%リトレースが理想（浅すぎ=だまし、深すぎ=転換リスク） |

**グレード:** S = 85点以上 / A = 70点以上 / B = 55点以上

デフォルトでは**Aグレード以上のみ通知**します（`InpMinScore=70`）。

## 監視パターンとトリガー

| パターン | 大きな流れ | 執行足 | 判定タイミング |
|---------|----------|--------|--------------|
| ① 4時間足レベルの押し目・戻り目 | 4H（ダウ理論） | 1H | 1H新バー確定毎 |
| ② 1時間足レベルの押し目・戻り目 | 1H（ダウ理論） | 15M | 15M新バー確定毎 |
| ③ 日足レベルの押し目・戻り目 | D1（ダウ理論） | 4H | 4H新バー確定毎 |
| ④-B 日足逆行の4H転換 | — | 4H | 情報通知のみ（「押し目/戻り目形成の起点の可能性」） |

**売りトリガーの例（パターン①）:**
1. 4時間足がダウ理論で下降トレンド（高値切り下げ＋安値切り下げ）
2. 1時間足で戻り目形成：直近スイング高値がその前の高値より低い（高値切り下げ）
3. 戻り目の起点となったスイング安値を**確定バーが初めて下抜けた瞬間**（安値更新＝下降転換確定）
4. → この瞬間に採点 → 規定スコア以上なら通知

## セットアップ

1. `FX4Pattern_Signal.mq5` を `MQL5\Experts\` にコピーしてコンパイル
2. **H1チャート**にアタッチ（内部で15M/1H/4H/D1を自動監視するため1チャートでOK）
3. スマホ通知を使う場合:
   - PC側: ツール → オプション → 通知タブ → MetaQuotes IDを入力
   - スマホ側: MT5アプリ → 設定 → チャットとメッセージ → MetaQuotes IDを確認
4. パラメータはデフォルトのままで運用開始可能

### 主要パラメータ

| パラメータ | 既定値 | 説明 |
|-----------|--------|------|
| `InpSwingDepth` | 3 | スイング判定の左右バー数（大きいほど大きな波のみ検出） |
| `InpMinScore` | 70 | 通知する最低スコア（70=A以上、55=B以上） |
| `InpOppForceVetoATR` | 1.0 | 反対勢力がATR×この値より近ければ却下 |
| `InpPushNotify` | true | スマホへのプッシュ通知 |
| `InpAutoTrade` | **false** | Sグレードでの自動エントリー（まずはOFFで通知のみ推奨） |
| `InpAutoMinScore` | 85 | 自動売買する最低スコア（S=85） |
| `InpSL_BufferATR` | 0.3 | SL = 直近スイング ± ATR×この値 |

## 推奨運用フロー

```
【フェーズ1】シグナル専用機として運用（自動売買OFF）
  → スマホに届いた通知を見て、チャートを自分の目で確認
  → マニュアル通り「自分で判断してエントリー」する練習
  → シグナルの精度を体感で検証

【フェーズ2】デモ口座で自動売買ON（Sグレードのみ）
  → InpAutoTrade=true / InpAutoMinScore=85
  → 1〜3ヶ月のデモ運用で成績を確認

【フェーズ3】少額リアル運用
  → InpRiskPercent=0.5 から開始
```

## バックテスト検証（合成データ・参考値）

`fx_signal_backtest.py`（リポジトリルート）で従来版と比較検証済み:

| システム | トレード数/3年 | 勝率 | 損益 | PF |
|---------|--------------|------|------|-----|
| 従来版（MAスロープ） | 174回 | 47.1% | +72R | 1.78 |
| 4P-Score（A以上） | 41回 | 46.3% | +16R | 1.73 |

**GOLDのみ（XAUUSD）:**

| システム | トレード数 | 勝率 | PF | 最大DD |
|---------|----------|------|-----|--------|
| 従来版 | 47回 | 42.6% | 1.48 | -8R |
| 4P-Score | 20回 | **50.0%** | **2.00** | -7R |

**読み方の注意:**
- 合成データ（幾何ブラウン運動）は実相場のサポレジ構造を持たないため、
  スイング・サポレジを使う4P-Scoreには**不利な検証環境**です。
  実データではより差が出る可能性があります（逆もあり得ます）
- 4P-Scoreはトレード数を1/4に厳選する設計です（マニュアルの
  「鉄板パターンのみ・厳選してトレード」の思想を反映）
- **実運用前に必ずMT5ストラテジーテスターで実データ検証してください**

実行方法:
```bash
python fx_signal_backtest.py        # Aグレード(70点)以上
python fx_signal_backtest.py 55     # Bグレード(55点)以上
python fx_signal_backtest.py 85     # Sグレード(85点)のみ
```

## 注意事項

- パターン④-C（大きなレンジでの転換）は強いサポレジの「強さ」の判断が
  目視に依存するため実装対象外です（④-Bの情報通知でカバー）
- 重要指標（FOMC・雇用統計・CPI）前後はスプレッド拡大のため
  シグナルが出ても見送り推奨
- GOLDはボラが大きいため `InpOppForceVetoATR` を 1.5 に上げるとより安全
- 過去の結果は将来の利益を保証しません。必ずデモ口座から始めてください
