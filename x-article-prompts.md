# X（旧Twitter）長文記事 作成プロンプト・テンプレート集

Claude Code / ChatGPT などで「バズりやすいX長文記事」を書くための4ステッププロンプト。
`{{ }}` の箇所を自分の情報に置き換えて使う。各ステップは順番に実行する（いきなり本文を書かせない）。

> ⚠️ 注意：このテンプレは「読まれる記事の作り方」を効率化するものであり、
> 「実行すれば必ず稼げる」ことを保証するものではない。成果には商品・導線・既存フォロワー等の前提が大きく影響する。
> 詳細は本リポジトリの検証メモ参照。

---

## 事前準備：ナレッジと素材

良い記事の差は「プロンプト」より「素材（一次情報）」で決まる。書く前に以下を用意する。

- **一次情報メモ**：自分の実体験・具体的な数字・失敗談・データ・独自の考え方
- **過去のセッション履歴**：Claude/ChatGPTでの過去のやり取り（思考や数値が詰まっている）
- **証拠スクショ**：成果（インプ数・売上・リスト獲得など）の管理画面キャプチャ

---

## ステップ1：テーマを決める（伸びるかどうかの8割がここで決まる）

### 1-A. リサーチ用プロンプト（任意・あくまで叩き台）

```text
あなたはX（旧Twitter）のトレンドリサーチャーです。
直近でバズっている長文記事（Article）を、日本国内と海外の両方から探してください。

【条件】
- 直近1〜3ヶ月以内に投稿された記事
- インプレッション・いいね・リポストが多いもの
- ジャンル：{{書きたい領域。例：AI、副業、マーケティング}}

【各記事について抽出する】
- 記事のタイトルと冒頭の3行
- 推定インプレッション数・エンゲージメント
- 何がフックになって伸びたのか（テーマ・切り口・感情）の分析
- 自分が書くとしたら、どの切り口を真似できそうか

【出力】
- 国内5本・海外5本をリストアップ
- それぞれ「なぜバズったか」を1〜2行で添える
- 最後に、今の自分が書くなら狙い目のテーマを3つ提案する
```

### 1-B. テーマ選びのチェックリスト（リサーチより重要）

- [ ] **旬のツール × 旬のプラットフォーム** の掛け算になっているか（どちらか古いと弱い）
- [ ] **具体的な金額・期間** が主役になっているか（「稼げます」はNG、「4日で42万」はOK）
- [ ] **再現性** が伝わるか（「これパクれます」と言い切れるレベル）
- [ ] **意外性・ツッコミどころ** があるか（「こんなにちょろくていいのか」）
- [ ] **自分が一次情報を語れる領域** か（借り物テーマは旬でも伸びない・バレる）

## ステップ2：見出し（構成案）だけを作る

```text
あなたはX（旧Twitter）の長文記事を書くプロのライターです。
これから書く記事の「見出し（構成案）」だけを作ってください。本文はまだ書きません。

【テーマ】
{{ステップ1で決めたテーマ}}

【素材】
- 次に貼る過去のセッション（会話履歴）を読み、記事に使える一次情報
  （自分の経験・数字・失敗・データ・考え方）を箇条書きで抽出してから構成を組む

【見出しのルール】
- 見出しはキャッチコピー。「〇〇まとめ」「なぜ〇〇か」のような説明文はNG
- 名詞で終わる、自然で素直な言い回しにする
- 数字を盛らない（「7選」と書くなら、本文で本当に7個出す）
- 読者が「おっ」と思って、次を読みたくなるかどうかで判断する

【構成の型】
1. 冒頭：結論かインパクトを即出し
2. 序文：背景を軽く
3. 本編：実績 → 仕組み → 具体的なやり方
4. CTA：末尾に誘導

【出力】
- 見出しを上から順に並べる
- 各見出しの下に、何を書くかを1行メモで添える
- まず3パターン出して、選べるようにする

---
（ここに過去のセッション履歴・一次情報メモを貼り付ける）
```

### 見出しの良し悪し判定（最終基準は「次が読みたくなるか」だけ）

- 説明文にしない（「実績まとめ」→ ✕）
- 口語のまま（「なんでThreadsなのか」「これ、パクれます」）
- 数字を入れる（「6つのAIエージェント」「意識した4つのこと」）
- 失敗をそのまま見出しに（「大事故その1：1分で10連投してアカウント凍結」）
- 疑問形で引っ張る（「なんで5%入れるだけで3万→42万になるのか」）

## ステップ3：本文を書く

```text
あなたはX（旧Twitter）の長文記事を書くプロのライターです。
さきほど作った見出しに沿って、本文を書いてください。

【素材として読み込むもの】
1. このあとに貼るX記事ナレッジ（書き方のルール）
2. 過去のセッション（会話履歴）。一次情報、つまり自分の経験・数字・失敗・データ・
   考え方を最優先で使う

■ 文体
- 敬語ベースだけど崩す。話し口調の敬語（「〜なんですよね」「〜ですけど」）
- 太字は使わない
- 文頭の「で、」は使わない（「それで、」にする）
- イコール記号は使わない。「つまり」「だから」で繋ぐ
- 「AI」と書ける場面でも具体名（Claude Code、Cursor、ChatGPT）を出す
- 括弧で本音・裏話感を出す

■ 一次情報
- 過去セッションから、自分にしか書けない経験・数字・失敗談を必ず入れる
- ネットで拾える情報の寄せ集めにしない

■ 離脱させないテンポ
- トーンの急ハンドル。真面目な解説の途中に、急にふざけた1文を入れる
- 「はい。」のような短文で脳に余白を入れる
- 読者を共犯者にする（「あなたは今ここまで読んでますよね？」）
- 読者が「え？」と思う前に、自分から先回りでツッコむ
- 促音で声を聞かせる（「クッッッソ」など）。1記事に2〜3箇所まで

■ 説得力
- 数字で殴ってから自虐で落とす（すごい数字 → 目標には届いてない → 自分で落とす）
- 数字は対比で見せる。ただし「2時間→10分」のような時間短縮の矢印表現はNG
- 失敗談を必ず入れる。他人は笑わず、自分を笑う

■ やってはいけないこと
- ですます調だけで最後まで書く／太字の多用／変なところで改行する
- 「〇〇って思った人、全然違います」のようなAIっぽい語りかけ
- 根拠のない断言

【書き方の手順】
1. 過去セッションから一次情報を抽出する
2. 見出しごとに本文を書く
3. 冒頭は最後にもう一度練り直す（最初の3行で「え？」を作る）

---
（ここにX記事ナレッジ本体 ＋ 過去のセッション履歴を貼り付ける）
```

## ステップ4：画像を用意する

```text
記事に入れる画像を用意してください。

【やること】
1. 数字の成果（インプ数・リスト獲得・売上画面など）はスクショで見せる
2. 仕組みやフローは図解で可視化する（HTMLで作ってスクショして画像化）
   - 黒背景に青アクセントの図解が映える
3. ツールの操作画面はスクショして、注目させたい箇所を赤枠で囲む

【出力】
- どの見出しの、どこに、どんな画像を入れるかを指定する
- 図解はHTMLで作って、スクショしやすい状態に整える

※Webサイトの自動スクショはスクレイピング・凍結のリスクがあるため、
　X記事への実際の貼り付け作業は手動で行う
```

### 画像の配置パターン

- **数字は必ずスクショ**で見せる（捏造と思われないため）
- **仕組みは図解**（HTMLで作成→スクショ）にすると一発で伝わる
- **長文が続いたら休憩用の画像**を挟んで目を休ませる

## クイック実行ガイド

1. 事前準備で「一次情報メモ」と「過去セッション」を集める
2. ステップ1で `{{書きたい領域}}` を埋め、チェックリストでテーマを確定
3. ステップ2で見出しを3パターン出させ、1つ選ぶ
4. ステップ3で本文を書かせる（ナレッジ＋過去履歴を必ず添付）
5. ステップ4で画像を指定・作成、貼り付けは手動で
